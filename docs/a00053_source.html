18// @file main_tasks.c
19// @ingroup sched-layer
20// @brief Contains the application specific tasks that are called regularly by the os_scheduler
21// @details Contains the application specific tasks that are called regularly by the os_scheduler
22// two different timings priorities are available:
23// 1. 1ms Tasks called from the os_scheduler interrupt
24// the jitter that you will have in the 1ms realtime tasks called by the interrupt depends
25// on other interrupts that have a higher interrupt priority
26// 2. 1ms, 10ms, 100ms, 1s Tasks called from the main loop
27// these tasks are for soft realtime and not for hard realtime
28// so in average they are called with the required timing but the jitter can be very huge,
29// depending on the calls before.
30// use this for your non-timing critical application state machines
31//
32// @note you might consider to implement a watchdog and put the watchdog triggering into some Task
33//
34// @version v1.0
35// @date 2019-08-09
36// @author M52409
37//
38//=======================================================================================================
